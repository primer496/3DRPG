# 急停与动画过渡经验总结

本文档记录从**走/跑急停 → 站立**过渡中遇到的问题和最终有效解决方案，供后续类似动画过渡参考。

---

## 一、问题现象

1. **抖动**：急停结束后回到站立时，角色或脚部出现明显抖动/抽搐
2. **位置偏移**：过渡到 Locomotion 时角色略微向前（或其它方向）偏移，两段动画的根位置不一致

---

## 二、有效解决方案（按优先级）

### 1. 代码与 Animator 分工

- **进入急停**：由代码控制（` moveInput` 松开 + 水平速度 > 阈值时 `CrossFade("StopType")`）
- **离开急停**：由 Animator 控制（在 StopType → Locomotion 设置 Transition，如 Exit Time 或 Speed 条件）
- **HSM 同步**：代码轮询 `GetCurrentAnimatorStateInfo(0).IsName("StopType")`，当 Animator 已切出 StopType 后，HSM 跟随切回 Move，避免逻辑与动画不同步

### 2. 动画片段导入：根变换设置

在 Animation Clip 的 **Inspector** 中（或 FBX 的 Animation 选项卡），调整：

| 设置项 | 建议值 | 说明 |
|--------|--------|------|
| **Root Transform Position (Y) - Based Upon** | **原始 (Original)** | 若为「质心 (Center of Mass)」，不同 pose 的质心位置不同，Stop 与 Idle 混合时根位置会跳 |
| **Root Transform Position (XZ) - Based Upon** | **原始 (Original)** | 同上，保证 Y 与 XZ 使用统一基准 |
| **Root Transform Rotation - Offset** | 按需微调（如 -4.37°） | 身体方向与 Stop 结束帧对齐，可逐度微调直到过渡自然 |

**要点**：Stop、Idle（及参与混合的 Locomotion 片段）尽量使用**相同的 Based Upon**，避免不同基准导致过渡时根位移。

### 3. Animator 状态与 Write Defaults

- **NormalMove**、**StopType** 的 `Write Default Values` 设为 **1**，避免未写入的骨骼通道在过渡时“残留旧值”造成抖动

### 4. Speed 衰减与交接

- 急停期间用**指数衰减**（而非 SmoothDamp）将 Animator 的 `Speed` 平滑回 0，避免极小时间常数时突变
- 若 Animator 用 Speed 条件过渡，代码需持续写入衰减后的 Speed，直至 Animator 完成过渡

### 5. 左右脚与脚步事件

- 用 **OnFootPlant** 动画事件（左脚=0、右脚=1）驱动 `StopFoot` 参数，使急停选用与步态一致的脚
- 在走路/跑步 clip 的脚着地帧添加事件，调用 `FootPlantAnimationEvents.OnFootPlant(int)`

---

## 三、无效或效果有限的做法

- **Transition Offset**：对“根位置差异”类偏移帮助不大
- **仅调 Transition Duration**：可改善混合观感，但无法消除基准不一致导致的位置跳变
- **基于质心的 Root Transform**：Stop 与 Idle 质心不同，混合时易产生前后偏移

---

## 四、推荐检查顺序

遇到类似「急停 → 站立」过渡异常时，可按此顺序排查：

1. **Root Transform Based Upon**：Stop 与 Idle 是否一致，优先改为 **原始**
2. **Write Default Values**：参与过渡的状态是否均为 1
3. **身体方向 Offset**：Root Transform Rotation 的 Offset 是否与 Stop 结束帧对齐
4. **代码与 Animator 分工**：进入由代码、离开由 Animator，HSM 只做同步
5. **Speed 衰减方式**：避免使用 SmoothDamp 极小时间常数，改用指数衰减并加帧时间下限
